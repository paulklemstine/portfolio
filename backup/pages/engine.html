Critical Implementation Details

B_j mod a Reduction

B_j = t_roots[j] * A_j * A_j_inv % a -- MUST reduce mod a. Omitting this causes silent errors in sieve initialization.

Sieve g(x) not Q(x)

Must sieve g(x) = ax² + 2bx + c where c = (b² - n)/a. Using Q(x) instead gives wrong smoothness thresholds.

Do NOT Reduce b mod a

Breaking incremental offset updates in Gray code polynomial switching. This subtle invariant cost days to debug.

LP Bound = min(B*100, B²)

B² gives large prime space too large for single large prime combining. B*100 is the sweet spot.

The 11 SIQS Optimizations

1. Knuth-Schroeppel Multiplier

K-S scoring to find optimal multiplier k for kN, up to 2.38x speedup. Improves factor base density for the sieve.

2. DLP Revival + LP Bound Fix

Double large prime combining with min(B*100, B²) bound. 30-50% more relations from partial-partial combining.

3. Singleton Filtering

Iterative column removal from the GF(2) matrix. 15-50% reduction in linear algebra work.

4. Multithread Sieve

multiprocessing.Pool with 2 workers. 1.77-1.81x speedup. Sieve is embarrassingly parallel across polynomials.

5. Bitpacked GF(2) Gauss Elimination

numpy uint64 bitpacking for the exponent matrix. Handles 80K rows in 1.2GB. 5x speedup over mpz-based Gauss.

6. Sieve Width Tuning

M reduced by 50% for better cache utilization. 1.34x at 60d. The sieve array was exceeding L2 cache.

7. C Lattice Sieve for GNFS

Special-q sieving in C with __int128 arithmetic. Enables GNFS at 50d+ where Python sieve was too slow.

8. C Trial Division

__int128 batch trial division in C extension. 35x faster than Python divmod for candidate verification.

9. DLP Rho Optimization

Small-prime trial division + rho limit 200 → 50. Fixed the 49.5% runtime hotspot in _quick_factor. ~4x reduction.

10. Int16 Sieve Array

Log scale 1024 → 64, array int32 → int16. Halves sieve memory (11MB → 5.7MB), 1.27x at 52d.

11. C Pollard Rho for _quick_factor

Wired existing pollard_rho_c.so into _quick_factor (was pure Python). 52d: 15.2s → 10.3s (1.47x). The .so existed but was not being used.

GNFS Pipeline

Phase 1: Polynomial Selection

Search ±1000 around m_0, score by norm size at typical sieve points. Degree: d=3 for <40d, d=4 for 40-65d, d=5 for 65-100d.

Phase 2: Factor Base & Sieve

Two-phase sieve: Phase 1 (b=1..5000, 2x area) for high yield at small b. Phase 2 (normal area, batch 500). C lattice sieve for speed.

Phase 3: Relation Processing

SLP combining (~50% of relations). SGE reduces matrix 21K → 15K (~30%). Singleton filtering for further reduction.

Phase 4: Linear Algebra

Bitpacked GF(2) Gauss elimination. Constrained by 12GB RAM + 8GB swap (practical limit ~5GB). Handles up to ~15K rows.

Phase 5: Square Root

Hensel lifting for algebraic square root. Low-skew polynomials can break Hensel sqrt -- poly selection must account for this.
